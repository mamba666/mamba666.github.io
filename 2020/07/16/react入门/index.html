什么是react

React 是一个声明式，高效且灵活的用于构建用户界面的 JavaScript 库。使用 React 可以将一些简短、独立的代码片段组合成复杂的 UI 界面，这些代码片段被称作“组件”。

Props/State/Forms属性和状态

Props

  • 组件就像一个函数，接受特定的输入（Props）,然后产生特定的输出。
  • V=f(Props)
  • Props是只读的，不能在组件中修改

State

  • 属性是外部传入的，所以不可改变。但是状态是组件私有的，可以改变。
  • 调用 this.setState()更新State，也是唯一的途径。
  • this.setState()使用方法：里面传入一个对象参数，这个对象就是this.state

Context介绍

什么是context

  • props属性是由上到下单向传递的
  • context提供了在组件中共享此类值的方法

来看看文档的描述：Context 提供了一个无需为每层组件手动添加 props，就能在组件树间进行数据传递的方法。

在一个典型的react应用中，数据大多是通过props从上而下进行传递的，但是队友某一些应用场景，如UI主题的切换，地区的偏好，这些属性是应用中大部分组件需要的。因此context就是为了解决这个问题而生的。Context 提供了一种在组件之间共享此类值的方式，而不必显式地通过组件树的逐层传递 props。

context的弊端

乱用context会使代码的复用性变差，如果只是为了避免层层传递pops，可以使用组件组合去代替。
