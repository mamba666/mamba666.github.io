jquey和vue的区别

最明显的区别就是，jquey是操作DOM，而vue则是操作数据。

vue.js专注于mvvm框架的vm（viewmodel）层，通过双向绑定，将视图和数据联系起来。从而可以通过对数据的操作就可以让视图发生改变。

vue的父子组件和兄弟组件如何传值

父–>子

在父组件中使用v-bind传递数据，在子组件中使用props接收数据，然后使用插值表达式使用数据
还有一个注意点，props是单向绑定的：即父可以传递数据影响子组件，但是不能反过来子组件影响父组件。这是为了防止子组件修改父组件的状态。所以不应该在子组件中修改 props 中的值，不然 Vue 发出警告。

子–>父

子组件通过this.$emit()的方式将值传递给父组件。然后父组件通过v-on去接收

vuex和axios的使用

vuex

概念

以可通过之前做的去哪儿，引申出来vuex的使用。

有一个场景，就是所在地的需求，即在城市选择页面选择的城市，会同步到首页，这是不同的组件，所以这里使用了vuex去管理这个数据。

还有两个例子:

  1. 比如说用户的登录信息。就跟微博一样，如果你不登陆，在所有页面上你所使用的功能都会受到限制，那么它们是如何知道你是否登录了呢？这就是多个视图需要依赖这个登录状态信息。
  2. 还是登录信息，假设你要退出登录，但是你登录的信息保存在登录页，但是退出需要到退出页面，这就是在另一个视图需要改变同一个登录状态信息。

es6里面const、let、var的区别

  • let 和 const都不能重复声明一个变量，但是var可以
  • let 和 const声明的变量，在块级内会存在暂时性死区，即它所声明的变量就“绑定”（binding）这个区域，不再受外部的影响。
  • var 和 let声明时，可以不赋值，但是const必须赋值

关于const的本质

const并非说值不能改变，而是变量指向的那个内存地址所保存的数据不能改动。
对于字符串、数值、布尔值这种简单的数据结构，就等同于常量
但是对于复杂的数据，比如对象、数组。变量指向的内存地址，保存的只是一个指向实际数据的指针，const 只能保证这个指针是固定的，至于它指向的数据结构是不是可变的，就完全不能控制了。

js里的垃圾回收机制

js是通过检测一个对象，是否还会被使用，假如确定不在使用了，那么就会清除这个内存。
一般常见的方法就是标记清除。即当变量进入环境时，就将这个变量标记为“进入环境”，反之就是“离开环境”
