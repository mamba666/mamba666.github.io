JS变量的过程

JS变量会有三个过程：创建create、初始化initialize 和赋值assign

这里以var的那个函数举例子

  1. 进入函数，为函数创建一个环境。
  2. 找到 fn 中所有用 var 声明的变量，在这个环境中「创建」这些变量（即 a 和 b）。
  3. 将这些变量「初始化」为 undefined。
  4. 开始执行代码

所以，在没有执行代码之前，使用var声明的变量即使被使用，也会被显示为undefined。

class

基本上，ES6 的class可以看作只是一个语法糖，它的绝大部分功能，ES5 都可以做到，新的class写法只是让对象原型的写法更加清晰、更像面向对象编程的语法而已。

使用原生实现class，可以发现class就是通过原型实现的

Object.getPrototypeOf()

这个方法是直接在原型对象上添加方法，以后的所有基于它的实例对象都会继承这个方法。

通过 下面的diamagnetic可以发现，Object.getPrototypeOf和proto作用是一样的。但是最好使用Object.getPrototypeOf代替proto。

因为proto并不是语言的本身特性，而是浏览器自带的，所以为了避免产生环境依赖，最好使用Object.getPrototypeOf代替之

export/import

模块功能主要由两个命令构成：export和import。export命令用于规定模块的对外接口，import命令用于输入其他模块提供的功能。

  • 通常情况下，export输出的变量就是本来的名字，但是可以使用as关键字重命名。
  • export输出的是动态实时的值，而之前的commonJS是缓存的值，不存在动态更新。
  • 在其他模块使用export定义接口之后，就需要import加载这个其他模块。

new Set

ES6 提供了新的数据结构 Set。它类似于数组，但是成员的值都是唯一的，没有重复的值。

假如从别人那里获取了一个新的对象，但是想添加一个新的方法，那么这个创建的名字很可能就会重复。如果有一种机制，保证每个属性的名字都是独一无二的就好了，这样就从根本上防止属性名的冲突。

Symbol是ES6引入的，是JS的第七种数据类型（前六种分别为undefined、null、Boolean、String、Object、Number）。

http&ajax

TCP/IP 三次握手和四次挥手

  • 三次握手
  1. 客户端向服务的发送SYN码数据包，表示客户端要向服务的建立连接
  2. 服务的收到客户端的连接请求后，会发送ACK码数据包给客户端，表示连接请求已经收到，再次向客户端确认是否需要真的简历连接
  3. 客户端在收到ACK码后会检验其是否正确。如果正确，则客户端会再次发送ACK码数据包给服务端，表示确立连接。
  4. 在以上三次握手都成功后，才会真的建立起连接，并且发送数据。
  • 四次挥手
  1. 当客户端发送完数据后，会给服务端发送FIN码数据包，表示数据已经传递完了
  2. 当服务的收到之后，会发生ACK码数据包给客户端，表示已经知道数据传送完了。客户端收到之后，就会关闭传送的通道
  3. 服务端会发送一个FIN，表示告知客户端数据已经发送完毕
  4. 当客户端收到FIN之后，会发送一个ACK。表示客户端已经知道数据发送完毕。然后客户端会关闭到传送的通道

http常用的状态码

  1. 2xx ：表示成功
  • 200 OK
  • 204 请求成功，但是服务端没有内容返回

  1. 3xx ：重定向

  2. 4xx ：客户端错误

  • 400 参数传递不当
  • 401 权限不够
  • 403 服务端已经理解了请求，但是拒绝响应
  • 404 客户端请求的资源或者数据不存在，一般原因有两个：写错了接口、服务端还没部署
  1. 5xx ：服务端错误
  • 500 服务端内部错误
  • 502 网关错误

浏览器从输入url到页面显示都发生了什么

大致的步骤：

  1. 首先根据域名通过DNS查询到IP
  2. 通过IP向服务器发起TCP连接
  3. 向服务器发起请求
  4. 服务器返回内容
  5. 浏览器解析渲染页面并且显示
  6. 关闭连接

http和https的区别

  1. https协议是由ssl+htpp协议构建的可进行加密传输、身份认知的网络协议，比http安全。
  2. http是超文本传输协议，而https是具有安全性的ssl加密传输协议
  3. 连接方式也不同，端口号也不同（http为80，https为443）

https：保护内容隐私；可确认对方的身份；防止内容篡改

浏览器的缓存（强缓存、协商缓存）

  • 什么是浏览器缓存呢？

简单来说，就是浏览器保存通过http获取到的数据，即将数据存储在本地（浏览器中）。

  • 缓存的资源去哪儿了呢?

memory cache：将资源缓存到内存，等待下次访问时，就直接从内存中获取。适合存放JS脚本、字体、图片等
disk cache：将资源保存到磁盘中，等待下次访问时，就直接从磁盘中获取。适合存放css文件等

  • 三级缓存原理（访问缓存的优先级）
  1. 现在内存中查询
  2. 再去磁盘中查询
  3. 再去网路请求
  4. 再将请求的数据缓存

get和post的区别

get：一般用域信息的获取
post：一般用于修改服务器上的资源

同源策略问题

  • 什么是源策略？
  1. 同源策略是JS的重要安全度量标准，主要是为了防止某个文档从多个不同源去装载。
  2. 同源策略指的是协议、端口、域名都要相同，缺一不可。
  • 为什么会有同源策略？

比如说，有一个坏人，将银行的登陆页面嵌入到他的山寨页面中，当用户输入真的用户名、密码后，坏人就可以通过JS读取到表单输入的内容。这样信息就被盗取了。

防抖函数&节流函数

之前一直不懂什么是防抖和节流。其实顾名思义，假设有一个需求，是鼠标停止时页面弹出alert，但是如果不进行防抖或者节流，鼠标只要一移动就会不断的发生弹出事件。

概念：

  • 函数节流: 频繁触发,但只在特定的时间内才执行一次代码
  • 函数防抖: 频繁触发,但只在特定的时间内没有触发执行条件才执行一次代码

可以发现，两者前提都是频繁的触发。节流就是只执行一次代码；防抖就是在某一个时间点内没有被触发时执行代码。

Vue

什么是生命周期

Vue实例从创建到销毁的过程就是生命周期

生命周期的作用

生命周期中由多个钩子函数，可以使得在控制vue逻辑时更加方便。比如说创建的时候要发生什么，更新的时候发生什么。

生命周期有几个阶段

总共有八个阶段

创建前后：beforeCreate、created
载入前后：beforeMount、mounted
更新前后：beforeUpdate、updated
销毁前后：beforeDestroy、destroyed

第一次页面加载会触发什么钩子函数

beforeCreate、created、beforeMount、mounted

DOM渲染在哪儿个周期中完成

mounted

生命周期分别适合什么场景

beforeCreate：可以在这里加上loading事件
created：可以在这里写loading结束事件，异步请求也适合在这里，比如请求数据
mounted：挂载元素，获取dom节点
boforeDestroy：可以做一个确认停止的确认框
nextTick：更新数据后立即操作dom

v-show和v-if的区别

v-show是css切换（display：none），适合频繁操作
v-if是操作dom（销毁和重新创建），适合在较少改变时使用

vue组件data为什么必须是函数

因为组件是可以复用的。JS里对象是引用的关系，假如data是一个对象，那么子组件就会互相污染。所以组件的data必须是函数。
